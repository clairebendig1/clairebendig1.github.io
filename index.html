Meta Reality Labs — Documentation Style Guide

Situation
When I joined Meta's Reality Labs in 2022, documentation across 25+ engineering teams was inconsistent — different terminology, different structure, different tone. There was no single reference for how to write at Reality Labs.
Task
Build a style guide that engineers and technical contributors would actually use — and that would hold up as the organization evolved.
Action
Gathered style materials from across the org, built the guide around developer and engineering audience needs, and drove adoption through a promotional video and an org-wide rollout.
Result
Three years later, that guide is integrated into Meta's automated Vale tooling and validated against every documentation diff published on the developer site. It became the adopted standard for all external documentation engineering teams at Meta.

Meta Reality Labs — Accessibility Writing Guidelines

Situation
Reality Labs had no shared accessibility standards for developer documentation. Accessibility was treated as an afterthought — added after content was written rather than designed in from the start.
Task
Establish accessibility as a core quality dimension for developer documentation, on par with accuracy and completeness.
Action
Became the accessibility champion for Reality Labs. Built the Bots4Docs AX writing guidelines command — an AI-assisted tool that checks content for accessibility issues at the point of authoring. Led accessibility reviews and brought standards back to the team.
Result
The AX command became part of the Bots4Docs toolkit — the top-used AI tools across Meta. Accessibility moved from a checklist item to a built-in quality gate in the documentation workflow.

Meta Horizon Developer Site — Essentials

Situation
The Horizon OS developer site needed a foundational content section that new developers could rely on to orient themselves to the platform. Information architecture was inconsistent, and teams lacked a shared framework for how to structure and contribute content.
Task
Build the information architecture, content standards, and contribution model that would make the Essentials section a reliable entry point for Horizon OS developers.
Action
Authored the Horizon OS Information Architecture Principles and Playbook — a formal IA framework enabling teams to make structure decisions independently. Built publishing guidelines, integrated design templates, and trained a new set of self-directed contributors.
Result
Launched not just new content but a new contribution model for the developer center. Showcase views increased 35% and GitHub documentation retention improved 12% following the initiative.
